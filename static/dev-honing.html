브랜치 내용 정리




브랜치 실습


main 브랜치로 이동하기: git checkout main

파일 당겨오기: git pull

새로운 브랜치 생성하기: git branch [브랜치명]

생성한 브랜치로 이동하기: git checkout [브랜치명]

생성된 html 파일을 편집하기(나: dev-honing.html)

편집을 완료하면 개인 브랜치로 커밋 푸시하기

각자의 작업이 끝나면 강사님께서 main 브랜치에 병합:
git checkout main으로 이동하고,
 git merge [브랜치명]으로 작업하실 것 같다.

브랜치 기본작업(생성과 전환)


브랜치 생성: git branch [브랜치명]

브랜치 목록 확인: git branch

브랜치 이름변경: git branch -m(--move) [기존 브랜치명] [변경할 브랜치명]

브랜치 삭제: git branch -d [브랜치명]

브랜치 전환: git checkout [브랜치명]


브랜치 병합


브랜치 병합 순서

1. 작업 브랜치로 전환: git checkout [작업 브랜치]

2. [브랜치명]을 [작업 브랜치]에 병합: git merge [브랜치명]

브랜치 병합 방법

1. Fast-Forward merge 방식
base가 같을 때에 사용하는 브랜치 병합 방법
상대적으로 뒤에 있는 브랜치가 앞에 있는 브랜치로 빠르게 이동해 병합하는 원리 따라서 새로운 커밋이 생기지 않는다
2. 3-Way merge 방식
base가 다를 때에 사용하는 브랜치 병합 방법
 base와 각 2개의 참조 커밋을 참고해서 병합함(총 3개를 참고) 새로운 커밋을 생성하고, 그 커밋을 '머지 커밋'이라고 한다
